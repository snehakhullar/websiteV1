Prototyping


God is in the details.

Mies van der Rohe

I primarily use three types of interactive prototyping techniques in my projects: (1) Browser - HTML & CSS, (2) Interactive Mock-ups- Invison, InDesign, and (3) Paper - hand-drawn sketches and mock-ups

I use interactive prototypes to better demonstrate interactions (using browser developer tools) such as transitions and hover states to the product development team. This enables me to provide product specifications to developers more easily and with greater clarity. I've learnt to design better interactions using new capabilities of HTML5 and CSS, such as transitions and background videos. I've done rapid prototyping directly on the browser to expedite design iterations.

Responsive maps

Time frame:June - August 2012

Role:UI designer & prototyper

Webpage link:Click here

This was a research project I conducted as a part of Esri innovation team in 2012. We created a boiler plate that can be used by developers to build responsive map-based applications.

Objective

Demonstrate that map-based applications can be made responsive

What I did (as a prototyper)

  • Defined break-points for different media queries
  • Wrote CSS code for generic visual design
  • Created three exemplary visual themes to demonstrate how the generic visual design can be customized
  • Tested the code on different devices (phones, tablets) and platforms (iOS & Android)

Impact

  • Presented this project at the Esri User Conference 2012, resulting in several hundreds of downloads of the boilerplate we created
Sage

Time frame:Feb - May 2013

Role:UX researcher, UX/UI designer, Prototyper

Sage is a mobile app for buying/selling home cooked meals that I conceptualized, designed and built during Grad school

Objective

Design and develop a product application to pitch a start-up idea to VCs

What I did (as a prototyper)

  • Defined product's development architecture based on modeler view controller (MVC) pattern
  • Chose Sinatra as the web application library to develop prototype since it is based on MVC pattern
  • Wrote SAAS code to implement visual design
  • Implemented a design iteration based on results of a usability study, that I did

Impact

  • The start-up pitch for the concept was very well received by the VCs.
  • Also, I presented the project during an exhibition at the Harvard School of engineer and sciences. We received great feedback on the simplicity and functionality of the app.
